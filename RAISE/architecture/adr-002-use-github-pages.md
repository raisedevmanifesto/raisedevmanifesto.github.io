# ADR-002: Use GitHub Pages for Site Hosting

**Status:** Accepted  
**Date:** 2025-05-15

## Context

We want to publish the RAISE Manifesto and related artifacts as a simple, static site, with all content and history open and accessible. We’ve chosen to host comments and discussion externally (e.g., Reddit, Substack).

## Decision

We will use GitHub Pages to host the RAISE Manifesto site. All site content will be managed in a public GitHub repository, making it easy to update, version, and share.

## Consequences

- The site is free to host, easy to update, and version-controlled.
- All artifacts (vision, values, user stories, ADRs) are transparent and open for review.
- No backend or server maintenance required.
- If we need more dynamic features in the future, we can revisit the decision.
